

# Naming Solutions, Projects, Pages, Widgets & Assets

Construct supports spaces, underscores and hyphens in Solution, Project, Page, Widget and Asset names. Assign informative and readable names that indicate use or purpose.

For example:

- Solution: Home

- Project: Entertainment

- Project: Lighting

- Project: Shades

- Page: Smart-TV

- Page: Audio

- Widget: Smart TV Controls

- Asset: Movies-1


## Best Practices

- Keep the names as short as possible to avoid file path problems while keeping them informative and readable.
  - For example: Software Lab 1, Software\_Lab\_1
- Use spaces between key words to increase readable.
  - For example: Access Panel, Access\_Panel
- Use hyphens and numbers for enumeration.
  - For example: Left Speaker-1; Left Speaker-2
- Avoid abbreviations unless they are commonly known:
  - For example: Meeting Rm-1. Mtg Rm-1 might be acceptable, but might be slightly more difficult to read immediately.

- Other than space, underscore and hypen--no other special characters are supported.


Page and Widget names **must** be entirely unique and distinct from each other; names **cannot** be similar to each other—even if they follow the above naming conventions.

For example, if an existing page is named Page1, a new page cannot be created and similarly named: Page-1, Page\_1 or Page 1.

Whereas, Solution, Project, and Assets names can be similar to each other.

For example, Solution-1, Solution 1; Project1, Project\_1; Asset 1, Asset-1 are allowed and supported.
