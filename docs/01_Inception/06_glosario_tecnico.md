# Glosario Técnico — AV-MCP Automator
**Fase RUP:** Inicio

| Término | Definición técnica | Contexto en este proyecto |
|---|---|---|
| **MCP (Model Context Protocol)** | Estándar abierto promovido por Anthropic que define cómo una aplicación provee contexto local seguro (archivos, herramientas, documentación) a un LLM mediante JSON-RPC 2.0. | El núcleo de comunicación entre Streamlit (UI) y Gemini, alojando las herramientas (tools) que ensamblan los archivos. |
| **FastMCP** | SDK de Python de alto nivel para construir servidores MCP usando decoradores simples. | Framework utilizado para exponer `search_tool`, `builder_tool` y `cuig_tool` como un servidor de contexto. |
| **Crestron CH5** | Framework JavaScript/HTML5 de Crestron para desarrollar interfaces táctiles avanzadas en sistemas de control AV profesional. Reemplaza la tecnología SmartGraphics. | Es el ecosistema final destino al que apunta el código de interfaz generado. |
| **Archivo `.cuig`** | Archivo de texto plano con extensión propietaria de Crestron Construct que representa una página de la interfaz, compuesto por bloques `{FileMetadata}`, `{Html}`, `{Css}`, `{PageAttributes}`. | Es el entregable primario del script Python `builder_tool.py`, diseñado para ser abierto y editado visualmente por el ingeniero AV. |
| **Archivo `.cuib`** | Archivo maestro de configuración del proyecto en Crestron Construct. Indexa todas las páginas (`.cuig`), rutas y configuraciones globales. | Archivo que es actualizado por `cuig_tool.py` cada vez que se genera una nueva página. |
| **LanceDB** | Base de datos vectorial embebida diseñada para búsqueda por similitud sobre embeddings, operando directamente sobre disco y requiriendo mínima memoria RAM. | Almacena y permite la búsqueda semántica de los esquemas JSON de los componentes CH5 y la documentación técnica de Crestron. |
| **Pydantic** | Librería de Python para validación de datos y gestión de configuraciones basada en anotaciones de tipos de Python. | Valida estrictamente el JSON generado por Gemini antes de enviarlo al ensamblador, evitando alucinaciones de parámetros o tipos de datos erróneos. |
| **Streamlit** | Framework web de código abierto en Python para la creación rápida de aplicaciones interactivas de datos y machine learning. | Interfaz de usuario (UI) principal donde el ingeniero AV ingresa el prompt en lenguaje natural y ajusta parámetros. |
| **Gemini Flash (2.5 Flash-Lite)** | Modelo LLM ligero y rápido de Google, disponible con alta cuota gratuita (1,000 llamadas/día). | Motor generativo responsable de razonar la distribución espacial y semántica a partir del prompt, devolviendo un layout en JSON. |
| **Builder_Tool** | Herramienta/función principal del servidor MCP encargada de ensamblar el archivo propietario. | Recibe el JSON validado de Pydantic y ejecuta el ensamblaje matemático (coordenadas a CSS) e inserción de atributos CH5 para crear el `.cuig`. |
| **Patrón Compilador MCP** | Arquitectura donde el LLM solo genera una estructura abstracta (JSON) y un script determinista (Python) la "compila" en el formato propietario final. | Estrategia central del proyecto para evitar que la IA invente sintaxis Crestron y asegurar 100% de compatibilidad con Construct. |
| **WYSIWYG** | "What You See Is What You Get". Paradigma de edición visual donde el resultado en pantalla es exactamente lo que se obtendrá. | Describe el IDE Crestron Construct, que requiere coordenadas absolutas en CSS para funcionar correctamente. |
| **Pro AV (Professional AudioVisual)** | Industria de integración de sistemas de audio, video y control para entornos corporativos, educativos o gubernamentales. | El sector de negocio de DACER S.A.C., el contexto donde este middleware aportará eficiencia de maquetación. |
| **Posicionamiento Absoluto CH5** | Técnica de diseño en Crestron Construct que requiere el uso estricto de coordenadas (`left`, `top`, `width`, `height`) en lugar de layouts fluidos como Flexbox. | Regla matemática que el Builder_Tool aplica para transpolar los campos `x, y, w, h` del JSON hacia el bloque `{Css}` del `.cuig`. |
| **Joins / Contract Editor** | Sistema de asignación de señales digitales, analógicas o seriales en Crestron que vincula lógicamente un elemento UI con el procesador físico. | Fuera del alcance del sistema; se asignan manualmente en Crestron Construct una vez que el layout base está generado. |
| **Web Xpanel** | Tecnología de Crestron que renderiza un proyecto CH5 en navegadores web de PC o tablet como un panel táctil virtual. | Herramienta utilizada por el ingeniero AV para previsualizar el panel final. |
| **PageAttributes** | Uno de los cuatro bloques del archivo `.cuig`, utilizado por el editor de Crestron para indexar y listar componentes internamente. | Bloque generado matemáticamente por el `builder_tool` para asegurar que las propiedades sean editables en el panel derecho de Construct. |
| **devicesvisited** | Atributo interno de Crestron inyectado en elementos HTML5 para control de estado o depuración. | Ejemplo de atributo `ccid_*` manejado por plantillas de Python y que la IA no debe generar ni conocer. |
| **ccid_*** | Prefijo de los atributos propietarios de Crestron inyectados en elementos HTML (`ccid_label`, `ccid_componenttype`, etc.). | Son manejados por las plantillas base en el código Python, previniendo alucinaciones y asegurando la compatibilidad de apertura del archivo `.cuig`. |
| **Embedding Vectorial** | Representación matemática (vector de números flotantes) de texto o datos, que captura significado semántico. | Formato en el que LanceDB guarda las descripciones de los componentes CH5 para encontrar el más adecuado al requerimiento del prompt. |
| **RAG (Retrieval-Augmented Generation)** | Técnica que mejora las respuestas de un modelo de lenguaje al proporcionarle información recuperada de una base de conocimiento externa. | El flujo que usa el AV-MCP: busca en LanceDB el esquema del componente (Retrieval) y lo envía a Gemini junto con el prompt (Augmented Generation). |
